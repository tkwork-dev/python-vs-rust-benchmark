# 要件定義書: Python vs Rust パフォーマンス比較検証

## 1. プロジェクト概要

### 1.1 目的
「Pythonは遅い」と言われる理由を定量的に検証するため、同一アルゴリズムをPythonとRustで実装し、実行時間を比較する。
さらに、Pythonの最適化手法（NumPy, lru_cache等）を活用した場合の高速化効果も検証する。

### 1.2 検証フェーズ

| フェーズ | テーマ | 検証ポイント | 状態 |
|---------|--------|-------------|------|
| 1 | フィボナッチ数列（再帰） | 関数呼び出しオーバーヘッド | ✅ 完了 |
| 2 | 素数計算（エラトステネスの篩） | ループ処理・メモリアクセスパターン | ✅ 完了 |
| 3 | 行列積（N×N） | CPU密な数値計算、ネストループ | ✅ 完了 |
| 4 | 文字列処理（回文判定） | オブジェクト生成・GCの影響 | ✅ 完了 |

---

## 2. 機能要件

### 2.1 ベンチマーク仕様

| フェーズ | アルゴリズム | 入力 | 出力 |
|---------|-------------|------|------|
| 1 | 素朴な再帰（メモ化なし） | N=40 | 計算結果、実行時間(ms) |
| 2 | エラトステネスの篩 | 上限=10,000,000 | 素数の個数、実行時間(ms) |
| 3 | 三重ループ行列積 | 200×200 | 検証値(C[0][0], C[199][199])、実行時間(ms) |
| 4 | 先頭末尾比較による回文判定 | 100万文字列 | 回文数、実行時間(ms) |

### 2.2 実装カテゴリ（3者比較）

| カテゴリ | 説明 |
|---------|------|
| Pure Python | 標準ライブラリのみ、素朴な実装 |
| Python + 最適化版 | NumPy, lru_cache, スライス等を活用した高速版 |
| Rust | 同一アルゴリズムのコンパイル済み実装 |

### 2.3 実装要件

#### Python側
- バージョン: Python 3.10以上
- 計測: `time.perf_counter`を使用
- 型ヒント: 使用する
- 依存パッケージ: `requirements.txt` で管理

#### Rust側
- エディション: 2021
- 外部クレート: 使用しない（標準ライブラリのみ）
- 計測: `std::time::Instant`を使用
- ビルドモード: `--release`（最適化あり）
- プロジェクト構成: Cargoワークスペース

### 2.4 出力フォーマット
各ベンチマークは以下の統一フォーマットで結果を出力する:

```
=== [テーマ名] ===
言語: [Python / Python (NumPy) / Rust]
[パラメータ情報]
[計算結果]
実行時間: XXX.XX ms
```

## 3. 非機能要件

### 3.1 公平性の担保
- 同一アルゴリズム・同一ロジックで実装する
- 言語固有の最適化（末尾再帰最適化など）は行わない
- Rustは`--release`ビルドで計測する
- 最適化版はアルゴリズム改善が含まれる場合はその旨を明記する

### 3.2 再現性
- 実行環境情報（OS、CPU、各言語バージョン）をREADMEに記載する
- 環境情報は `docs/environment/` に保存する
- 他の開発者が同一手順で再現できること

### 3.3 計測精度
- 高精度タイマーを使用する
- データ生成等の前処理は計測対象外とする

### 3.4 依存管理
- Python: `requirements.txt` による管理
- Rust: Cargoワークスペースによる一元管理

## 5. 実行手順

### セットアップ
```bash
# Python依存パッケージのインストール
pip install -r requirements.txt
```

### 一括実行
```powershell
# PowerShellから実行
powershell -ExecutionPolicy Bypass -File run_benchmark.ps1

# または run.bat をダブルクリック
run.bat
```

### 個別実行

#### Pure Python
```bash
python python/fibonacci.py
python python/sieve.py
python python/matrix.py
python python/palindrome.py
```

#### Python + 最適化版
```bash
python python/fix/fibonacci_lru.py
python python/fix/sieve_numpy.py
python python/fix/matrix_numpy.py
python python/fix/palindrome_regex.py
python python/fix/comparison_all.py    # 統合比較
```

#### Rust
```bash
cd rust
cargo build --release
./target/release/fibonacci.exe
./target/release/sieve.exe
./target/release/matrix.exe
./target/release/palindrome.exe
```

## 6. 期待される結果

### Pure Python vs Rust
- フィボナッチ: Rustが50〜60倍高速（関数呼び出しオーバーヘッド）
- 素数計算: Rustが40〜50倍高速（ループ処理コスト）
- 行列積: Rustが80〜100倍高速（数値演算の累積オーバーヘッド）
- 回文判定: Rustが20〜30倍高速（文字列操作・GC）

### Python 最適化版 vs Rust
- 適切な最適化手法を活用すれば、Rustと同等の性能が出るケースもある
- 特にNumPyの行列演算はBLAS実装によりRustの素朴な実装と同等以上

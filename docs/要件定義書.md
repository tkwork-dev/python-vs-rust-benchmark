# 要件定義書: Python vs Rust パフォーマンス比較検証

## 1. プロジェクト概要

### 1.1 目的
「Pythonは遅い」と言われる理由を定量的に検証するため、同一アルゴリズムをPythonとRustで実装し、実行時間を比較する。

### 1.2 検証対象（フェーズ1）
フィボナッチ数列の再帰計算による**関数呼び出しオーバーヘッド**の比較検証。

### 1.3 今後の検証予定（フェーズ2〜4）
| フェーズ | テーマ | 検証ポイント |
|---------|--------|-------------|
| 2 | 素数計算（エラトステネスの篩） | ループ処理・メモリアクセスパターン |
| 3 | 行列積（N×N） | CPU密な数値計算、ネストループ |
| 4 | 文字列処理（回文判定） | オブジェクト生成・GCの影響 |

---

## 2. 機能要件

### 2.1 ベンチマーク仕様（フィボナッチ数列・再帰）

| 項目 | 内容 |
|------|------|
| アルゴリズム | 素朴な再帰（メモ化なし） |
| 入力 | フィボナッチ数 N（デフォルト: N=40） |
| 出力 | 計算結果、実行時間（ミリ秒） |
| 計測方法 | 計算処理のみを計測対象とし、起動時間は含めない |
| 実行回数 | 1回（N=40の再帰は十分な計測時間を確保できるため） |

### 2.2 実装要件

#### Python側
- バージョン: Python 3.10以上
- 外部ライブラリ: 使用しない（標準ライブラリのみ）
- 計測: `time`モジュールの`perf_counter`を使用
- 型ヒント: 使用する

#### Rust側
- エディション: 2021以上
- 外部クレート: 使用しない（標準ライブラリのみ）
- 計測: `std::time::Instant`を使用
- ビルドモード: `--release`（最適化あり）で計測

### 2.3 出力フォーマット
両言語で以下の統一フォーマットで結果を出力する:

```
=== フィボナッチ数列 再帰計算 ===
N: 40
結果: 102334155
実行時間: XXX.XX ms
```

---

## 3. 非機能要件

### 3.1 公平性の担保
- 同一アルゴリズム・同一ロジックで実装する
- 言語固有の最適化（メモ化、末尾再帰最適化など）は行わない
- Rustは`--release`ビルドで計測する（デバッグビルドでは不公平）

### 3.2 再現性
- 実行環境情報（OS、CPU、各言語バージョン）をREADMEに記載する
- 他の開発者が同一手順で再現できること

### 3.3 計測精度
- 高精度タイマーを使用する（Python: `perf_counter`、Rust: `Instant`）
- ウォームアップは不要（再帰計算自体が十分な実行時間を持つため）

## 5. 実行手順（想定）

### Python
```bash
python python/fibonacci.py
```

### Rust
```bash
cd rust/fibonacci
cargo build --release
./target/release/fibonacci.exe
```

### 一括実行
```powershell
./run_benchmark.ps1
```

---

## 6. 期待される結果

- Pythonに対してRustは**50〜100倍以上**高速になると予想される
- 主因: Pythonの再帰呼び出しごとにフレームオブジェクトが生成され、動的型チェックが走るため
- Rustはコンパイル時に型が確定し、関数呼び出しもスタック操作のみで完了する
